## 起動

```bash
npm run admin
# → http://127.0.0.1:3000 が起動（外部IPからは接続不可）
```

`.env` に `YOUTUBE_API_KEY` が設定されている必要がある（動画手動追加に使用）。

ブラウザで `http://localhost:3000` または `http://127.0.0.1:3000` を開く。

- [ ] ページが読み込まれる
- [ ] ヘッダーに「🛠 管理画面（localhost）」が表示される

## ② 🎨 件数不足ハイライト

- [ ] 動画数が **0〜4 件**のジャンルが **赤の件数ピル** で表示される

- [ ] 動画数 8 件以上は通常色のピル

## ③ ▶ バッチ実行 → 承認 / 差し戻し

> 注意：実バッチは YouTube API を呼ぶ（クォータ消費 約 2,900 unit）

- [ ] 「▶ バッチ実行」ボタンを押すと進捗メッセージが表示される
- [ ] 数十秒で完了し、`data/videos.draft.json` が生成される
- [ ] 完了後、ページ上部の「バッチ結果プレビュー」に **新規追加 / 更新** の差分一覧が表示される
- [ ] 「✅ draft 承認」を押すと確認ダイアログ → OK → `videos.json` にマージ
- [ ] マージ後の `videos.json` で **manual 動画が消えていないこと**（最重要）
- [ ] あるいは「❌ draft 差し戻し」で draft が破棄される

## ④ ⭐ 動画手動追加（manual）

- [ ] 「動画手動追加」フォームに **YouTube URL**（`https://www.youtube.com/watch?v=...`）を貼る → videoID 抽出
- [ ] **短縮URL**（`https://youtu.be/...`）も抽出される
- [ ] **11文字 videoID 直接入力**も受け付ける
- [ ] 不正入力（10文字など）はエラーメッセージで弾かれる
- [ ] カテゴリ・ジャンルを選択 → 「⭐ manual 動画として追加」
- [ ] 追加後、動画一覧の該当ジャンルに **`manual` バッジ付き** で表示される
- [ ] バッチ実行後も manual 動画が残っていることを確認

## ⑤ 🗑 動画削除・並び替え

- [ ] 各動画行の「削除」ボタンで動画が消える（確認ダイアログ）
- [ ] 「↑」「↓」ボタンで動画の並び順が変わる
- [ ] order フィールドが自動再付与される

## ⑥ 🚫 ブロックリスト

- [ ] 「ブロックリスト」セクションに現在の channelId / videoId 一覧が表示される
- [ ] 種別を選んで ID を入力 → 「追加」で `categories.json` に保存される
- [ ] 各 ID の「削除」ボタンで該当 ID が消える
- [ ] 次回バッチ実行時、ブロックリストの動画/チャンネルが draft に含まれない

## ⑦ 🔒 セキュリティ（NS-010）

- [ ] `http://127.0.0.1:3000` でアクセスできる
- [ ] **同一LANの別端末から `http://<このマシンのIP>:3000` にアクセス**して接続できない or 403 が返ることを確認
  - 接続できない理由：`app.listen(port, '127.0.0.1')` で外部バインド禁止
  - 403 になるシナリオ：Host ヘッダ検証で `localhost`/`127.0.0.1`/`::1` 以外を拒否
